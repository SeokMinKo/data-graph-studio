# PRD: Data Table Enhancement (Sorting & Searching)

## 1. 목표
데이터 테이블의 정렬(Sorting)과 검색(Searching) 기능을 강화하여 대용량 데이터에서도 빠르고 직관적인 사용자 경험을 제공한다.

## 2. 배경
현재 Data Graph Studio의 데이터 테이블에서 다음 문제가 발생하고 있음:
- **Sorting 문제**: 헤더 클릭 시 정렬이 작동하지 않음. `setSortingEnabled(True)` 설정되어 있으나 `PolarsTableModel.sort()` 메서드가 구현되어 있지 않음.
- **Searching 문제**: 기본 검색은 작동하나, UX가 부족함 (디바운싱 없음, 검색 중 피드백 없음, 검색 결과 카운트 없음).

## 6. 데이터 구조

### 6.1 정렬 상태 (PolarsTableModel 내부에 저장)
```python
@dataclass
class SortState:
    column: Optional[str] = None  # 정렬 컬럼
    order: Qt.SortOrder = Qt.AscendingOrder  # 정렬 순서
    sort_indices: Optional[pl.Series] = None  # pl.Series(dtype=pl.Int32) - 메모리 효율
    # 캐시 무효화 조건:
    # - 새 DataFrame 설정 시
    # - 다른 컬럼 정렬 시  
    # - 외부에서 clear_sort() 호출 시
```

### 6.2 검색 상태 (TablePanel에서 관리)
```python
@dataclass  
class SearchState:
    query: str = ""
    case_sensitive: bool = False
    selected_columns: Optional[List[str]] = None  # None = 전체, [] = 선택 없음
    result_count: int = 0
    is_searching: bool = False
    history: List[str] = field(default_factory=list)  # max 5
```

### 6.3 검색 + 정렬 적용 순서
1. 원본 데이터에 **검색(filter)** 먼저 적용
2. 검색 결과에 **정렬(sort)** 적용
3. 필터와 검색은 별개 (필터 → 검색 → 정렬 순서)

## 7. 성능 & 메모리 요구사항

### 7.1 정렬 최적화
- Polars의 `sort()` 메서드 활용 (C++ 기반, O(n log n), **unstable sort**)
  - stable sort 필요 시 `maintain_order=True` 옵션 사용 (약간 느림)
- `nulls_last=True` 옵션으로 NULL 처리
- **정렬 인덱스만 캐싱** (`_sorted_df` 대신 `sort_indices: pl.Series`)
  - 캐시 무효화: 새 DataFrame 설정, 다른 컬럼 정렬, clear_sort() 호출 시
- 원본 인덱스는 `with_row_index()` 사용하여 추적
- 데이터 접근 시 인덱스 기반 on-demand 조회

### 7.2 검색 최적화
- Polars의 `str.contains()` 활용 (벡터화, SIMD)
- 검색 대상 컬럼 수 제한 (기본 20개)
- 디바운싱으로 불필요한 검색 방지 (300ms)
- 검색 결과는 필터링된 DataFrame 반환 (Polars는 내부적으로 최적화)

### 7.3 메모리 관리
- 정렬 인덱스: `pl.Series(dtype=pl.Int32)` 사용 → 100만 행 = ~4MB
- 검색 결과: 원본 DataFrame의 필터링된 뷰 (추가 복사 없음)
- 컬럼 캐시는 정렬 시 무효화 (정렬된 순서로 재구축)
- 메모리 증가 제한: 원본 데이터의 **10% 이하** (기존 20% → 10%로 강화)

### 7.4 대용량 데이터 처리
- **100,000행 이상**: QThread에서 정렬 실행 (UI 블로킹 방지)
- **500,000행 이상**: 정렬 시 프로그레스 바 표시
- **1,000,000행 이상**: 정렬 예상 시간 표시 + 취소 버튼
- 윈도우 모드에서는 현재 윈도우 내에서만 정렬/검색
