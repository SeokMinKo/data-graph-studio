## Task 2: Narrow except blocks in `comparison_engine.py` and `filtering.py`

**Files:**
- Modify: `data_graph_studio/core/comparison_engine.py:429,537`
- Modify: `data_graph_studio/core/filtering.py:709`

### Fix A: comparison_engine.py:429 (statistical test)

The try block runs scipy statistical tests (e.g. `scipy.stats.ttest_ind`, `scipy.stats.mannwhitneyu`). These raise `ValueError` for bad inputs (too few samples, all-zero arrays), `TypeError` for wrong dtypes, and `ArithmeticError` (including `FloatingPointError`) for numerical edge cases. The handler stores the error in `result["error"]` and returns gracefully — this is intentional.

```python
# Before (line 429):
except Exception as e:
    result["error"] = str(e)
    logger.error("comparison_engine.statistical_test_failed", extra={"error": e})

# After:
except (ValueError, TypeError, ArithmeticError) as e:
    result["error"] = str(e)
    logger.error("comparison_engine.statistical_test_failed", extra={"error": e})
```

### Fix B: comparison_engine.py:537 (correlation)

Same pattern — scipy correlation methods raise `ValueError`, `TypeError`, `ArithmeticError`.

```python
# Before (line 537):
except Exception as e:
    result["error"] = str(e)
    logger.error("comparison_engine.correlation_failed", extra={"error": e})

# After:
except (ValueError, TypeError, ArithmeticError) as e:
```

### Fix C: filtering.py:709 (_apply_single_filter)

The try block applies a Polars filter expression. Polars raises `InvalidOperationError` for type incompatibility, `ComputeError` for execution failures, `ColumnNotFoundError` for bad column names, `TypeError`/`ValueError` for Python-level type errors. The handler re-raises as `QueryError` — intentional conversion.

```python
# Before (line 709):
except Exception as e:
    raise QueryError(
        "Filter execution failed",
        operation="_apply_single_filter",
        context={"column": str(f.column), "operator": str(f.operator.value), "value": str(f.value)},
    ) from e

# After:
except (
    pl.exceptions.InvalidOperationError,
    pl.exceptions.ComputeError,
    pl.exceptions.ColumnNotFoundError,
    TypeError,
    ValueError,
) as e:
    raise QueryError(
        "Filter execution failed",
        operation="_apply_single_filter",
        context={"column": str(f.column), "operator": str(f.operator.value), "value": str(f.value)},
    ) from e
```

Verify that `pl` (polars) is already imported at the top of `filtering.py` (it should be).

### Regression tests to add

Create `tests/unit/test_constants_regression.py`:

```python
"""Regression tests for constants and API fixes from Round 10.

Ensures DEFAULT_SCHEME_NAME value is stable and LazyFrame schema
access does not emit PerformanceWarning.
"""
import warnings
import polars as pl
import pytest
from data_graph_studio.core.constants import DEFAULT_SCHEME_NAME


def test_default_scheme_name_value():
    """DEFAULT_SCHEME_NAME must equal 'Page' — changing it would break FilteringManager state."""
    assert DEFAULT_SCHEME_NAME == "Page"


def test_default_scheme_name_is_string():
    assert isinstance(DEFAULT_SCHEME_NAME, str)


def test_filtering_manager_uses_default_scheme_name():
    """FilteringManager should create a default scheme named DEFAULT_SCHEME_NAME."""
    from data_graph_studio.core.filtering import FilteringManager
    fm = FilteringManager()
    assert DEFAULT_SCHEME_NAME in fm.get_schemes()


def test_collect_schema_no_performance_warning():
    """data_query_helpers should use collect_schema() not .schema on LazyFrame."""
    from data_graph_studio.core.data_query_helpers import compute_eager_column_stats
    df = pl.DataFrame({"a": [1, 2, 3], "b": [4.0, 5.0, 6.0]})
    lf = df.lazy()
    with warnings.catch_warnings():
        warnings.simplefilter("error", pl.exceptions.PerformanceWarning)
        # Should not raise PerformanceWarning
        result = compute_eager_column_stats(df["a"])
    assert result is not None
```

Note: The `collect_schema()` test exercises the path indirectly via `compute_eager_column_stats`. If `data_query_helpers` is called differently, adjust the test to trigger the LazyFrame schema access path.
